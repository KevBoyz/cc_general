Transforme o arquivo analysis.ipynb em uma série de arquivos .py, um para cada questão do arquivo AV3-CD-IA.pdf (confirme a quantidade de questões lendo o PDF; não assuma um número fixo).

Nomeação: salve os arquivos como q1.py, q2.py, ..., qN.py, na raiz do diretório de saída.

Mapeamento célula -> questão: associe cada célula de código do notebook à questão do PDF que ela responde. Se uma célula de setup (imports, carregamento de dados, pré-processamento) for usada por mais de uma questão, repita essa célula em cada arquivo que dependa dela, para que cada .py seja executável de forma independente, sem depender de rodar outro arquivo antes. Se atente ao fato de que um preprocessamento nos dados é realizado, os dados devem manter o mesmo estado no notebook que nos arquivos, então para fazer de forma organizada, crie funções para preprocessar agilmente no arquivos. Todos os outputs devem ser identicos ao notebook

Conteúdo a remover: descarte todas as células markdown. Não adicione comentários novos que não existiam no notebook original; comentários já existentes no código podem ser mantidos.

Saída de figuras: toda figura gerada (matplotlib, seaborn, etc.) deve ser salva com plt.savefig() na pasta image/qx/ (x = número da questão), e não apenas exibida com plt.show(). Use os.makedirs("image/qx", exist_ok=True) no início de cada arquivo que gera figura, para criar a pasta automaticamente caso não exista. Dê nomes de arquivo descritivos para cada figura (ex.: image/q3/histograma_idade.png), evitando sobrescrever figuras quando uma questão gera mais de uma.

Outros outputs: qualquer output que no notebook aparecia por exibição implícita de célula (ex.: célula tipo df.head() ou variável solta) deve ser convertido em print() explícito no .py, preservando a mesma informação que aparecia no notebook (incluindo prints de shapes, métricas, resultados de modelos, etc.).

Fidelidade: cada arquivo .py deve produzir, ao ser executado isoladamente, exatamente as mesmas informações (texto e figuras) que a(s) célula(s) correspondente(s) produziam no notebook — nada a mais, nada a menos.

Ao final na criação dos arquivos, valide estaticamente se o estado das variaveis, e outputs se manterá identico ao notebook
